# API Reference

MORPH exposes a REST API (Fastify), a WebSocket endpoint for real-time updates, and a JSON-RPC endpoint for direct MCP protocol access.

## REST Endpoints

### System

| Method | Path                 | Description                              |
| ------ | -------------------- | ---------------------------------------- |
| `GET`  | `/api/version`       | Version info                             |
| `GET`  | `/api/health`        | Server health and MCP connection summary |
| `GET`  | `/api/config`        | Current active configuration             |
| `PUT`  | `/api/config`        | Update full configuration                |
| `POST` | `/api/config/reload` | Reload config from disk                  |
| `POST` | `/api/config/import` | Import config from Claude/VS Code format |

### MCP Servers

| Method   | Path                      | Description                      |
| -------- | ------------------------- | -------------------------------- |
| `GET`    | `/api/mcps`               | List all MCP servers with status |
| `GET`    | `/api/mcps/:name`         | Get MCP server details + tools   |
| `GET`    | `/api/mcps/:name/tools`   | List tools for a specific MCP    |
| `POST`   | `/api/mcps/:name/restart` | Disconnect and reconnect an MCP  |
| `POST`   | `/api/mcps`               | Add a new MCP server dynamically |
| `PUT`    | `/api/mcps/:name`         | Update an existing MCP server    |
| `DELETE` | `/api/mcps/:name`         | Remove an MCP server             |

### Per-MCP Tool Calls

| Method | Path             | Description                              |
| ------ | ---------------- | ---------------------------------------- |
| `POST` | `/api/mcp/:name` | Call a tool on a specific MCP (JSON-RPC) |

### Logs

| Method | Path               | Description                                           |
| ------ | ------------------ | ----------------------------------------------------- |
| `GET`  | `/api/logs`        | Query logs (params: `mcp`, `level`, `since`, `limit`) |
| `GET`  | `/api/logs/:id`    | Get a single log entry by ID                          |
| `GET`  | `/api/logs/stream` | SSE stream of new log entries                         |

### Stats

| Method | Path                      | Description                           |
| ------ | ------------------------- | ------------------------------------- |
| `GET`  | `/api/stats`              | Live aggregate metrics snapshot       |
| `GET`  | `/api/stats/toon`         | TOON conversion stats                 |
| `GET`  | `/api/stats/toon/history` | TOON savings history (param: `since`) |
| `GET`  | `/api/calls/totals`       | Call totals (param: `since`)          |
| `GET`  | `/api/calls/totalizers`   | Aggregate token totals                |

### OAuth

| Method | Path                   | Description                   |
| ------ | ---------------------- | ----------------------------- |
| `GET`  | `/api/oauth/authorize` | Initiate OAuth authorize flow |
| `GET`  | `/api/oauth/callback`  | OAuth redirect callback       |

### MCP Protocol (Direct)

| Method | Path   | Description                                 |
| ------ | ------ | ------------------------------------------- |
| `POST` | `/mcp` | JSON-RPC MCP protocol (direct agent access) |
| `GET`  | `/mcp` | SSE stream for MCP notifications            |

## WebSocket Channels

Connect at `ws://localhost:3101/ws`. Messages are JSON with the format:

```json
{
  "channel": "logs|stats|health|config",
  "event": "event_type",
  "data": {},
  "timestamp": "2025-01-01T00:00:00.000Z"
}
```

| Channel  | Events                      | Data                            |
| -------- | --------------------------- | ------------------------------- |
| `logs`   | `tool_call`                 | New log entry with TOON savings |
| `stats`  | `savings_update`            | Aggregate TOON savings metrics  |
| `health` | `connected`, `disconnected` | MCP server name                 |
| `config` | `reloaded`                  | Full updated config object      |

Clients can send `{"channel": "ping"}` to receive a `{"channel": "ping", "event": "pong"}` response.

## JSON-RPC Format

Tool calls use the standard MCP JSON-RPC format:

### Request

```json
{
  "jsonrpc": "2.0",
  "method": "tools/call",
  "params": {
    "name": "tool_name",
    "arguments": { "key": "value" }
  },
  "id": 1
}
```

### Response (success)

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "TOON formatted result...",
        "_meta": {
          "originalTokens": 250,
          "toonTokens": 120,
          "savings": "52.00%"
        }
      }
    ]
  }
}
```

### Response (error)

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32603,
    "message": "Internal error",
    "data": { "details": "Tool 'unknown' not found" }
  }
}
```

## Error Format

API errors follow a consistent format:

```json
{
  "error": "Human-readable message",
  "code": "ERROR_CODE"
}
```

| HTTP Status | Error Code         | Description                           |
| ----------- | ------------------ | ------------------------------------- |
| `400`       | `INVALID_INPUT`    | Invalid request body                  |
| `401`       | `UNAUTHORIZED`     | Basic Auth required                   |
| `404`       | `NOT_FOUND`        | Resource not found                    |
| `404`       | `MCP_NOT_FOUND`    | MCP server not found                  |
| `404`       | `TOOL_NOT_FOUND`   | Tool not found                        |
| `409`       | `CONFLICT`         | Resource conflict                     |
| `409`       | `ALREADY_EXISTS`   | MCP already exists                    |
| `422`       | `VALIDATION_ERROR` | Schema validation failed              |
| `422`       | `INVALID_CONFIG`   | Invalid configuration                 |
| `422`       | `ENV_MISSING`      | Required environment variable missing |
| `500`       | `INTERNAL_ERROR`   | Unexpected server error               |

## OAuth Endpoints

### GET /api/oauth/authorize

Redirects the user to the MCP server's authorization page.

### GET /api/oauth/callback

Handles the OAuth redirect callback. Receives the authorization code and exchanges it for tokens via the PKCE flow.

---

## Per-MCP Endpoint

### POST /api/mcp/:name

Direct JSON-RPC access to a specific backend MCP's tools. Accepts and returns standard MCP JSON-RPC messages.

```bash
curl -X POST http://localhost:3101/api/mcp/demo-stdio \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": { "name": "ping", "arguments": {} },
    "id": 1
  }'
```
