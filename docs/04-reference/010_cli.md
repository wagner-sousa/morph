# CLI Reference

MORPH ships as a single binary (`morph`) with two commands: `start` and `import`.

## Usage

```
morph <command> [options]
```

## Commands

### `morph start`

Start the MORPH hub (default command).

| Option         | Short | Type                       | Default                                    | Description                                                   |
| -------------- | ----- | -------------------------- | ------------------------------------------ | ------------------------------------------------------------- |
| `--config`     | `-c`  | path                       | `./morph.json` or `$MORPH_CONFIG`          | Path to `morph.json` (settings)                               |
| `--mcp-config` |       | path                       | sibling `.mcp.json` or `$MORPH_MCP_CONFIG` | Path to `.mcp.json` (MCP servers)                             |
| `--port`       | `-p`  | number                     | from config                                | Web UI port override                                          |
| `--transport`  |       | `stdio\|http`              | `stdio`                                    | Agent-facing transport                                        |
| `--log-level`  |       | `debug\|info\|warn\|error` | from config                                | Log level override                                            |
| `--mcp`        |       | string                     | —                                          | Start in single-MCP mode, exposing only the named MCP's tools |
| `--validate`   |       | boolean                    | —                                          | Validate config and exit                                      |
| `--version`    |       | boolean                    | —                                          | Show version and exit                                         |
| `--help`       |       | boolean                    | —                                          | Show help text                                                |

#### Examples

```bash
# Start with default config (stdio transport)
morph start

# Start with HTTP transport on custom port
morph start --transport http --port 8080

# Validate config only
morph start --validate

# Start in single-MCP mode (only expose demo-stdio tools)
morph start --mcp demo-stdio

# Start with debug logging
morph start --log-level debug

# Config path override
morph start -c /etc/morph/production.json
```

### `morph import`

Import MCP server configurations from other tools into `.mcp.json`.

| Option      | Type                   | Default       | Description                                       |
| ----------- | ---------------------- | ------------- | ------------------------------------------------- |
| `--from`    | path                   | required      | Config file to import                             |
| `--format`  | `claude\|vscode\|auto` | `auto`        | Source format (auto-detect if not set)            |
| `--merge`   | path                   | `./.mcp.json` | Target `.mcp.json` to merge into (by server name) |
| `--dry-run` | boolean                | —             | Print result to stdout without writing            |

#### Examples

```bash
# Import from Claude Desktop config (writes ./.mcp.json)
morph import --from ~/.config/Claude/claude_desktop_config.json

# Import from VS Code workspace into a specific .mcp.json
morph import --from .vscode/mcp.json --merge ./.mcp.json

# Preview an import without writing
morph import --from other/.mcp.json --dry-run

# Specify format explicitly
morph import --from config.json --format claude
```

## `--mcp` Flag

The `--mcp <name>` flag restricts MORPH to expose tools from only one backend MCP server. This is useful for:

- **Sandboxing** — ensure an agent can only access specific servers
- **Testing** — isolate a single MCP for debugging
- **Reduced complexity** — avoid overwhelming agents with too many tools

When `--mcp` is set, the agent-facing MCP server only lists and routes tools belonging to the named MCP. The full config still applies for the Web UI — all MCPs are visible and manageable in the dashboard.

## Environment Variables

| Variable                 | Default             | Description                             |
| ------------------------ | ------------------- | --------------------------------------- |
| `MORPH_CONFIG`           | `./morph.json`      | Path to settings file (`morph.json`)    |
| `MORPH_MCP_CONFIG`       | sibling `.mcp.json` | Path to MCP servers file (`.mcp.json`)  |
| `MORPH_DATA_DIR`         | `./data`            | SQLite database directory               |
| `MORPH_TRANSPORT`        | `stdio`             | Agent transport: `stdio` or `http`      |
| `MORPH_SHUTDOWN_TIMEOUT` | `10000`             | Graceful shutdown drain timeout in ms   |
| `MORPH_WEB_USERNAME`     | —                   | Web UI Basic Auth username              |
| `MORPH_WEB_PASSWORD`     | —                   | Web UI Basic Auth password              |
| `CORS_ORIGIN`            | —                   | Allowed CORS origin (production)        |
| `NODE_ENV`               | —                   | Set to `production` to disable dev CORS |

## Exit Codes

| Code | Meaning                                                        |
| ---- | -------------------------------------------------------------- |
| `0`  | Success                                                        |
| `1`  | General error (invalid config, unknown command, runtime error) |
| `2`  | Validation error (`--validate` with invalid config)            |
